## 简介

`tools/client_gui.py` 是 ESP32 BLE 设备管理器的图形界面客户端，基于 Flet 框架开发，提供直观易用的图形界面，支持设备扫描连接、信息查看、OTA 升级、WiFi 配网、LED 控制、自定义命令等功能。

**版本**: 1.3.1

**界面特性**:
- 固定窗口大小 1280×800
- 响应式按钮状态（加载中/禁用）
- 实时进度显示
- 双日志面板（系统日志 + BLE 传输日志）
- 深浅色主题自动切换
- PIN 码认证输入
- ASCII/HEX 自定义命令发送与定时循环发送

---

## 环境要求

- **操作系统**: Windows 10/11、macOS 10.15+、Linux
- **Python**: 3.8 或更高版本
- **蓝牙**: 支持 BLE（蓝牙 4.0+）的适配器
- **固件**: 设备端需烧录 ble_srv v1.3.1 或兼容版本固件
- **依赖库**: flet >= 0.20.0, bleak >= 0.20.0

## 设备连接

### 扫描设备

1. 应用启动后自动开始扫描（默认超时 5 秒）
2. 点击左侧面板「扫描」按钮可手动重新扫描
3. 扫描超时可在下拉菜单选择：3秒/5秒/10秒/15秒
4. 扫描期间按钮显示「读取中」并禁用其他按钮
5. 使用 `BleakScanner` 回调方式扫描，支持 Windows 和 macOS 跨平台兼容

**扫描按钮状态**:
- 空闲：显示「扫描」文字和扫描图标
- 扫描中：显示旋转 Loading 图标，文字变为「读取中」，按钮禁用
- 完成：恢复正常状态

**跨平台注意**:
- Windows: 使用 WinRT 蓝牙后端
- macOS: 使用 CoreBluetooth 后端，RSSI 从扫描广告数据中直接获取

### 设备过滤

在过滤输入框中输入关键词，可实时过滤设备列表（按设备名称前缀匹配）。

### 选择设备

1. 在左侧设备列表中点击要连接的设备
2. 设备列表显示：设备名称、MAC 地址、RSSI 信号强度
3. RSSI 参考：
   - -30 ~ -50 dBm: 信号极好
   - -50 ~ -70 dBm: 信号良好
   - -70 ~ -90 dBm: 信号一般

### PIN 码认证

如果设备启用了应用层认证（`CONFIG_BLE_SRV_AUTH_ENABLED`）：
1. 在左侧面板「PIN」输入框中输入设备密码
2. 连接成功后客户端会自动发送 PIN 认证作为第一个 GATT 命令
3. 认证失败设备会主动断开连接

### 连接/断开

1. 选中设备后点击顶部「连接」按钮
2. 连接成功后按钮文字变为「断开」，状态徽章显示绿色「已连接」
3. 连接中按钮显示加载状态
4. 点击「断开」按钮断开当前连接

**连接流程**:
1. 清理旧连接（如有）
2. 等待 300ms 让 BLE 栈稳定
3. 建立新连接（使用 `use_cached=False` 避免缓存问题）
4. 连接成功后自动发送 PIN 认证（如启用）
5. 认证通过后自动读取设备信息

## 设备信息页

设备信息页包含四个信息板块，点击「刷新」按钮重新读取：

### 设备综合信息

显示内容：
- 芯片型号
- MAC 地址
- 固件版本
- CPU 频率
- 芯片温度（如支持）
- 运行时间（Uptime）
- 上次重启原因

### 内存信息

三大类内存信息（紧凑单行显示）：
- **堆内存**: 总大小 / 空闲 / 最小空闲
- **内部内存**: 总大小 / 空闲 / 最小空闲
- **SPI RAM**（如支持）: 总大小 / 空闲 / 最小空闲

### CPU 信息

双行紧凑显示：
- 第一行：CPU 频率 | 核心数 | 芯片 Revision | IDF 版本
- 第二行：运行时间 | 任务数量 | CPU 使用率

### Flash 信息

- Flash 总大小
- Flash 空闲大小
- Flash 速度（MHz）
- 分区数量
- 当前运行分区名称

**刷新按钮**:
- 点击后按钮显示「读取中」和 Loading 图标
- 所有其他按钮暂时禁用
- 读取完成后自动恢复

## LED 控制页

### 颜色控制

1. **RGB 滑块**: R（红）、G（绿）、B（蓝）三个滑块，范围 0-255
2. **数值显示**: 每个滑块右侧显示当前数值（十进制）
3. **颜色预览**: 左侧显示颜色预览色块，随滑块实时变化
4. **颜色代码**: 预览色块下方显示十六进制 RGB 颜色代码（如 #FF0000）
5. **应用颜色**: 调整好颜色后自动发送或点击应用（实时发送）

**默认值**: 所有滑块默认值为 0（LED 关闭状态）。

### LED 开关

- 「打开 LED」按钮：点亮 LED
- 「关闭 LED」按钮：熄灭 LED

### 特效控制

1. **特效选择下拉菜单**:
   - 无（none）
   - 呼吸灯（breath）
   - 闪烁（blink）
   - 彩虹（rainbow）
   - 频闪（strobe）

2. **速度滑块**: 调整特效速度，范围 1-255，默认 50
   - 数值越小，特效变化越快
   - 数值越大，特效变化越慢

3. **应用特效按钮**: 选择特效和速度后点击应用

**注意**: 颜色通道顺序为 RGB，不是 GRB。WS2812 灯带顺序可在固件 menuconfig 中配置。

## WiFi 页

### 连接 WiFi

1. 在 SSID 输入框输入 WiFi 名称
2. 在密码输入框输入 WiFi 密码（开放网络可留空）
3. 点击「连接」按钮
4. 按钮显示「读取中」状态，等待连接结果

### WiFi 状态

点击「刷新状态」按钮查看当前 WiFi 状态：
- 连接状态（已连接/未连接）
- SSID 名称
- IP 地址
- 信号强度（RSSI）

### 断开 WiFi

点击「断开」按钮断开当前 WiFi 连接。

### 清除凭据

点击「忘记网络」按钮清除 NVS 中保存的 WiFi 密码，重启后不会自动连接。

### NTP 同步

点击「NTP 同步」按钮触发网络时间同步（需要 WiFi 已连接）。

**注意**: URL OTA 必须先连接 WiFi 才能使用。

## OTA 升级页

OTA 页支持两种升级方式和完整的状态反馈。

### 蓝牙 OTA (BT OTA)

1. **选择固件文件**:
   - 点击「选择文件」按钮浏览并选择 `.bin` 固件文件
   - 或手动在输入框中输入固件路径
   - 选择后自动解析固件版本号和大小

2. **开始升级**:
   - 点击「蓝牙 OTA」按钮开始升级
   - 按钮显示「读取中」状态，其他按钮禁用
   - 进度条实时显示传输进度
   - 状态文字显示当前阶段和速度、剩余时间

3. **传输过程**:
   - 状态显示：检查固件 → 接收固件中 → 校验中 → 应用中
   - 进度条显示百分比和传输速度
   - BLE 传输日志面板显示详细的传输日志
   - 使用 12 包滑动窗口 ACK 协议保证可靠性

4. **升级完成**:
   - 显示「OTA升级成功」
   - **重要提示**: 每次OTA建议重启设备，OTA失败的话需要重启设备再次OTA
   - 设备会在 3 秒后自动重启

### URL OTA

1. 在 URL 输入框输入固件下载地址（HTTP/HTTPS）
2. 确保设备已通过 WiFi 页连接 WiFi
3. 点击「URL OTA」按钮开始
4. 设备从 URL 下载固件并升级
5. 进度条显示下载进度

### 默认 URL OTA

点击「默认URL」按钮使用固件中内置的默认 URL 进行 OTA（需固件配置支持）。

### 中止 OTA

- OTA 进行中时「中止」按钮可用
- 点击中止按钮发送中止命令
- 设备清理后进入 ABORTED 状态
- 300ms 后自动重置为 IDLE 状态
- **中止后建议重启设备**

### OTA 状态说明

| 状态 | 说明 |
|------|------|
| 空闲 | 就绪，可以开始 OTA |
| 检查固件 | 正在验证固件版本和大小 |
| 检查通过 | 版本检查通过，开始传输 |
| 无需更新 | 版本相同或更旧，拒绝升级 |
| 接收固件中 | 正在传输固件数据 |
| 校验中 | 传输完成，正在 CRC 校验 |
| 校验通过 | CRC 校验成功 |
| 校验失败 | CRC 校验失败，固件损坏 |
| 应用中 | 正在写入 OTA 分区 |
| 应用成功 | 升级成功，设备即将重启 |
| 应用失败 | 应用失败 |
| 正在中止 | 正在中止 OTA |
| 已中止 | OTA 已中止 |
| 错误 | 发生错误 |

**错误码说明**:
- `0x01` 无效命令
- `0x02` 固件大小无效
- `0x03` Flash 写入失败
- `0x04` 无可用 OTA 分区
- `0x05` 固件校验失败
- `0x06` 设备内部错误
- `0x07` 设备忙
- `0x08` 网络未连接（URL OTA）
- `0x09` 用户中止
- `0x0A` 连接断开
- `0x0B` 远程固件版本更旧（拒绝降级）
- `0x0C` 固件版本相同（无需更新）
- `0x0D` CRC 校验失败

## 自定义命令页

自定义命令页提供与设备进行原始数据通信的能力，方便第三方应用扩展功能。

### 通讯日志

上半部分为深色背景的通讯日志显示区：
- 显示发送（TX，青色）和接收（RX，琥珀色）的数据
- 每条日志包含时间戳、方向标记、数据长度和内容
- 定时自动发送的数据标记为 `AUTO TX`
- 支持 ASCII 和 HEX 两种显示格式
- 日志上限 500 条，自动滚动

### 命令输入

下半部分为命令输入区域：
- **输入框**: 占一半宽度，支持回车键快速发送
- **发送按钮**: 点击发送当前输入内容
- **格式切换**: ASCII / HEX 两种输入模式
  - ASCII 模式: 直接输入文本，按 UTF-8 编码发送
  - HEX 模式: 输入空格分隔的十六进制数，如 `01 02 03 0A`

### 定时发送

- **开关**: 启用/禁用定时循环发送
- **间隔输入**: 发送间隔，单位毫秒，默认 2000ms，范围 10-60000ms
- 定时发送通过主线程事件循环执行，避免异步冲突
- 断开连接时自动停止定时发送

### 使用示例

**发送 ASCII 文本**:
1. 确认格式为 ASCII
2. 在输入框输入 `hello`
3. 点击发送或按回车

**发送 HEX 数据**:
1. 切换到 HEX 格式
2. 在输入框输入 `01 02 03`
3. 点击发送

**定时循环发送**:
1. 输入要发送的内容
2. 勾选「定时发送」
3. 设置间隔时间（如 1000ms）
4. 数据会按设定间隔自动发送并记录到日志

## 按钮状态说明

所有操作按钮遵循统一的交互规范：

### 正常状态
- 可点击，显示操作图标和文字

### 加载状态
- 按钮图标变为旋转 Loading 动画
- 文字变为「读取中」（或对应操作提示）
- 按钮被禁用，不可点击
- **所有其他按钮也暂时禁用**，防止并发操作

### 完成/超时
- 操作完成后自动恢复正常状态
- 如果超时（可配置），自动恢复并提示
- 失败时显示错误信息

### 连接状态
- 未连接：显示「连接」，绿色调
- 已连接：显示「断开」，红色调
- 连接中：显示 Loading，禁用

## 常见问题

### Q: 启动时提示缺少依赖？

**A**: 执行完整安装：
```bash
pip install flet>=0.20.0 bleak>=0.20.0
```

### Q: 启动后窗口空白或闪退？

**A**:
1. 使用 `--debug` 模式启动查看详细错误
2. 检查 Python 版本是否 >= 3.8
3. 尝试升级 flet: `pip install --upgrade flet`
4. Windows 上可能需要安装 WebView2 Runtime（Windows 10/11 通常已内置）

### Q: 扫描不到设备？

**A**:
1. 确认设备已上电并正在广播 BLE-SRV-XXXXXX
2. 确认电脑蓝牙已开启
3. Windows: 设置 → 隐私和安全性 → 蓝牙，允许桌面应用访问蓝牙
4. macOS: 系统设置 → 隐私与安全性 → 蓝牙，允许终端或 Python 访问
5. 尝试靠近设备，减少 2.4GHz 干扰（WiFi、微波炉等）
6. 增加扫描超时时间到 10 秒或 15 秒
7. 如果设备列表为空但 CLI 客户端能扫描到，尝试重启 GUI 应用或重新安装 bleak 库

### Q: 连接失败？

**A**:
1. 确认设备未被其他设备连接（BLE 通常只支持单连接）
2. 尝试重启设备蓝牙或重启设备
3. Windows 可尝试在设备管理器中禁用再启用蓝牙适配器
4. 重启 BLE Device Manager 应用
5. 靠近设备，确保信号强度 > -70dBm

### Q: OTA 进度长时间停滞？

**A**:
1. BLE 传输日志会显示"设备接收进度停滞"警告
2. 自动重传机制会处理丢包，耐心等待
3. 如果频繁停滞：
   - 关闭其他蓝牙设备（耳机、鼠标等）
   - 靠近设备，避免遮挡
   - 暂停大流量 WiFi 下载
4. 如果超时过多，会自动中止，重启设备后再试

### Q: OTA 后设备不重启或工作异常？

**A**:
1. **每次 OTA 建议重启设备**
2. OTA 失败必须重启设备后再次 OTA
3. 如果 OTA 完成但设备没重启，可手动点击「重启」
4. 如果设备变砖（无法连接），需要通过串口重新烧录固件

### Q: URL OTA 按钮点击后提示错误？

**A**:
1. URL OTA 必须先连接 WiFi
2. 切换到 WiFi 页，连接 WiFi 并确认获取 IP
3. 检查 URL 是否正确（必须是 http:// 或 https:// 开头）
4. 确认设备能访问该 URL（同局域网内可搭建本地 HTTP 服务器）

### Q: LED 颜色显示不对？

**A**:
1. 确认使用 RGB 顺序（RRGGBB）
2. FF0000 = 纯红，00FF00 = 纯绿，0000FF = 纯蓝
3. 如果红绿相反，检查固件中 WS2812 的颜色顺序配置
4. 滑块默认值为 0（黑色/关闭），不是上次设置值

### Q: 连接新设备后 OTA 日志不更新？

**A**:
1. 此问题已通过通知代际计数器（_notify_gen）修复

## 注意事项

### 启动前准备

1. 首次使用请安装所有依赖：`pip install flet bleak`
2. Windows 系统确保蓝牙已开启并授权应用访问
3. macOS 系统需要在系统设置中允许终端访问蓝牙
4. Linux 系统需要运行 bluez 蓝牙服务

### 连接管理

5. 应用启动时自动扫描设备，无需手动点击扫描
6. 设备断开连接时会自动重置所有 UI 状态
7. 连接新设备前会自动清理旧连接并等待 BLE 栈稳定
8. 窗口大小固定为 1280×800，不支持手动缩放

### OTA 升级重要提示

9. **每次 OTA 建议重启设备**，确保新固件完全加载
10. **OTA 失败必须重启设备后再次尝试**，不要连续重试
11. OTA 过程中不要关闭应用或断开蓝牙
12. OTA 过程中不要让电脑进入睡眠状态
13. 蓝牙 OTA 时建议关闭其他蓝牙设备以减少干扰
14. URL OTA 需要设备先连接 WiFi
15. 版本相同或更旧时会拒绝 OTA，这是正常的版本保护机制
16. OTA 成功后设备会在 3 秒后自动重启，蓝牙会断开

### LED 控制

17. RGB 滑块默认值为 0（全黑/LED关闭状态）
18. 颜色通道顺序为 RGB，修改滑块后实时发送
19. 特效速度数值越小变化越快（1=最快，255=最慢）
20. 设置特效后颜色可能被特效覆盖，关闭特效即可恢复纯色

### WiFi 操作

21. WiFi 密码保存在设备 NVS 中，重启后自动连接
22. 使用「忘记网络」清除保存的凭据
23. URL OTA 前必须在 WiFi 页确认已连接并获取 IP 地址

### 日志与调试

24. BLE 传输日志显示详细的 OTA 进度和重传信息
25. 系统日志记录所有操作结果和错误
26. 使用 `--debug` 模式启动可清除 Python 缓存，解决代码更新不生效问题
27. 遇到问题时保存日志内容便于排查

### 安全与恢复

28. OTA 有 CRC32 校验机制，不会刷入损坏的固件
29. 建议保留串口烧录能力以备设备变砖时恢复
30. 不要在 OTA 过程中断电或手动复位设备
